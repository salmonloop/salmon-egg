Start Composer Directions

首页输入区要像进入 AI cockpit 的启动页，同时保证展开/收起的生命周期闭环。

Shared lifecycle spine
State 1
Collapsed

默认首页。英雄建议卡可见，输入区保持克制。

State 2
Primed

输入框获得焦点，容器放大，展开“任务预设/Agent/上下文”辅助区。

State 3
Interacting

焦点在输入框、Agent 下拉、附加组件、建议项之间移动时保持展开。

State 4
Submitting

发送后锁定输入态，保留结构但压低动画与交互，避免抖动和误收起。

State 5
Reset / Return

外部点击、导航离开、会话创建完成后回到明确状态，而不是悬挂中间态。

A. Docked Hangar
Salmon Egg
英雄内容退到背景，底部输入区像机库控制台，展开时向上打开。

Docked Hangar

最贴近当前 Start 页结构。输入区固定在底部，聚焦后像驾驶舱控制台向上开启，风险最低。

B. Lifted Cockpit
Salmon Egg
进入任务舱前的预备页

Lifted Cockpit

输入区从底部抬升到视觉中心，更有仪式感，但导航回弹、焦点恢复和空态切换会更复杂。

C. Mission Console
Salmon Egg
输入区是主舞台，右侧挂载任务辅助组件，像完整 chat 页的前置控制台。

Mission Console

信息更强，但会把首页从“启动页”推向“小型工作台”，如果把握不好会太像 chat 页复制版。

推荐方向

  • A. Docked Hangar 最适合先落地。仪式感够强，又能复用当前 Start 页和完整 chat 页的结构关系。
  • 它最容易承载完整生命周期：外部点击收起、Agent 下拉保持展开、发送中冻结、进入 chat 后干净退出。

设计重点

  • 不要只做“放大一点”这种弱变化，要做成“英雄区退后，控制台升起”的空间感。
  • 展开态里的辅助组件要控制数量，只保留和首次提问强相关的 2-3 类内容。
